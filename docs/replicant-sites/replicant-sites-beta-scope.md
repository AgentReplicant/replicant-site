# Replicant Sites — Beta Client Scope Note

**Audience:** First beta client (the barber friend, and future first beta-tier clients).

**Tone:** Friendly, plain-language, non-legal. This is not a contract — it's a clear note so we both know what's included and what isn't. It exists so neither of us gets a weird surprise three weeks in.

---

Hey —

Excited to build your site. Before we start, I want to make sure we're on the same page about what's included so there are no awkward surprises later. Here's the short version.

## What this is

This is a **beta project** for Replicant Sites — meaning you're one of the first businesses I'm building for. In exchange, I'm waiving my build fee. You get a professional website. I get to use the project to refine the process and (with your permission) show it as an example.

## What's included

- One professional website for your business
- Sections like: hero, services, gallery, booking or contact CTA, about, reviews, location/hours, policies, FAQ
- Mobile-first design — looks good on phones, since that's where most of your customers will land
- We'll use your existing booking platform (Booksy, Square, Calendly, etc.) and link to it from the site
- Domain setup and connection (you'll own the domain in your name)
- Basic SEO so your business name shows up correctly on Google
- 1–2 rounds of revisions after the first draft
- Launch + a check-in a week after to make sure everything's running

## What's not included

To keep this scope honest:

- **Custom booking software.** We'll use what you already have — building a booking system from scratch is a separate project.
- **Customer accounts or logins.** Same reason.
- **Custom payment systems.** Your existing booking platform handles payments. We're not building a new one.
- **Logo design.** If you have a logo, great. If not, we can either use a clean text-based treatment or you can have one designed separately and add it later.
- **Unlimited revisions.** We get 1–2 revision rounds after the first draft. After that, additional changes can be discussed as a separate small project.
- **Ongoing content updates after launch.** Once the site is live, you (or someone you hire) handle ongoing content changes. If you ever want me to manage updates, we can set up a maintenance arrangement separately.

## What you'll cover

The build fee is waived, but a few real-world costs are still yours:

- **Domain registration** (usually $10–20 per year)
- **Hosting or platform fees** (if any apply — most of what we'll use is low-cost or free)
- **Your booking platform's fees** (Booksy, Square, etc. — whatever you're already paying)
- **Any paid stock photos, custom fonts, or third-party tools** we agree on together before we use them

Nothing here is hidden. If a cost comes up, I'll tell you before we spend a dollar.

## What I'm asking in return

Since this is a beta build, I'd love permission to:

- Use the finished site in my portfolio
- Use screenshots of the site in marketing materials
- Mention your business name when showing the work
- Ask you for a short testimonial after launch, only if you're happy with the result
- Show before/after examples (if you had a previous site or social-only presence)

You already answered these in the intake checklist — this is just restating them so we both have it in writing.

## Timeline

Once your intake checklist is filled out **and** your photos and materials are gathered, I'll start the build. From there, expect:

- First draft ready in **1–2 weeks**
- Revisions and final polish in another **week or so**
- Launch shortly after final approval

No assets, no build — so the faster you send things over, the faster we launch.

## How we'll communicate

- Best for quick stuff: text or DM
- Best for longer stuff (revisions, content, big decisions): email
- I'll send you the staging URL when the first draft is ready so you can review on your own time

If anything's unclear, ask me. Better to clear it up now than mid-build.
